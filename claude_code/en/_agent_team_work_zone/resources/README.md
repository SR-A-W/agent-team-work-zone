# resources/ — non-member resources

This directory holds shared resources **that do not belong to any specific agent workstation** — skills, general-purpose custom subagents, role archetype references, install scripts, reserved hooks, etc.

## Why `resources/` and not mixed into workstations

- Stored at the **same level** as flat workstations / team workstations, physically isolated to avoid conceptual confusion
- All sources that `bootstrap.sh` needs to install into `.claude/` live under the `skills/` and `agents/` subdirectories
- Source-of-truth rule: **only edit sources under `resources/`**; those under `.claude/` are derived copies and must not be edited by hand

## Source-of-Truth Rule

- **Source**: `resources/{skills, agents, role_archetypes, scripts, hooks}/`
- **Runtime**: `.claude/skills/` and `.claude/agents/` (synced by bootstrap)
- **Do not** directly edit `.claude/skills/*` or `.claude/agents/*`
- The English version `claude_code/en/_agent_team_work_zone/` is generated by Translator from zh after zh stabilizes; it is not an independent source

## Development Workflow

1. Edit `resources/skills/<name>/SKILL.md` (or `resources/agents/<name>.md`)
2. Run `bash resources/scripts/bootstrap.sh` to sync into `.claude/`
3. Restart the Claude Code session or use `/agents` to refresh so it loads the new version
4. Test
